Tutorial: Bouw de 'Zon en Zekerheid' Website (Conceptueel)

Hoi! Deze tutorial helpt je op weg met het bouwen van de 'Zon en Zekerheid' website. We richten ons op het *begrijpen* van de stappen, de juiste HTML-structuur (semantiek) en hoe je CSS (vooral Flexbox) gebruikt voor layout en styling. We werken mobile-first: eerst voor kleine schermen, dan voor grotere.

Belangrijke Regels & Tips (Lees dit goed!)

  • Jouw Code: Schrijf de HTML en CSS zélf. Deze tutorial geeft richting, geen kopieerbare code.
  • AI Hulp: Gebruik AI voor *tekstcontent* is prima. Kleine code *snippets* genereren ter inspiratie mag, maar je moet alles kunnen uitleggen. Begrijp je je eigen code niet in de video? Dan is het fraude = onvoldoende.
  • Geen Kopieerwerk: Code van anderen overnemen = plagiaat = onvoldoende.
  • Assets: Zoek zelf rechtenvrije afbeeldingen/video's (bv. Pexels). Geen placeholders.
  • Eindproduct: Werkende, responsive site (.zip) + video (max 5 min) met uitleg van je code (structuur, CSS-keuzes, responsive aanpak).
  • Focus: Semantische HTML, Flexbox voor layout, Mobile-First CSS.

Stap 0: De Basis Setup

Zet je projectstructuur op:

  1. Maak een projectmap (bv. zon-en-zekerheid).
  2. Maak hierin: index.html, blog.html, style.css, en een map images.
  3. Zet de minimale HTML-basisstructuur in beide .html bestanden. Denk aan: 
    Definieer het documenttype (<!DOCTYPE html>).
Start het HTML-document (<html lang="nl">).
Maak een <head> sectie met:
    Karakterset (<meta charset="UTF-8">).
    Viewport instellingen voor responsiviteit (<meta name="viewport" ...>).
    Een passende <title>.
    Een link naar je CSS-bestand (<link rel="stylesheet" href="style.css">).
Maak een lege <body> sectie waar de content komt.
Sluit <html> af.
  4. Controleer of de <link> naar style.css correct is.

Nu zijn we klaar voor de homepage: index.html.

Stap 1: Homepage (`index.html`) - De Header

Dit deel bevat je logo en de hoofdnavigatie.

HTML Structuur (Concept)

Denk semantisch! Wat is de functie van dit blok?

Gebruik het <header> element voor de bovenste balk van je pagina.

Plaats daarbinnen een <div> met een class (bv. `container`) om de inhoud te groeperen en later de breedte te beheren.

In de container, maak je een element voor je logo (dit kan een <div> zijn, of direct een <img> als je een afbeelding gebruikt). Geef het een class (bv. `logo`).

Voeg ook een <nav> element toe voor de navigatielinks; dit is semantisch correct.

Binnen de <nav>, gebruik een ongeordende lijst (<ul>) voor de menu-items.

Elk menu-item is een lijstitem (<li>) met daarin een link (<a>).

Geef de "Boek nu" link een extra class (bv. `btn btn-primary`) zodat je deze als knop kunt stylen. Zorg dat de "Blog" link naar `blog.html` wijst.

CSS Styling (Concept)

Begin met de basisstijlen die altijd gelden (mobile-first).

Globale Styles:
Reset browser defaults voor `margin` op de `body`.
Stel een basis `font-family` in.
Definieer je `.container` class: geef het een procentuele `width` (bv. 90%) voor flexibiliteit, een `max-width` voor grote schermen, en `margin: 0 auto` om te centreren.

Header Styles (Mobile):
Geef de `header` wat `padding` boven/onder en misschien een `border-bottom`.
Style het logo (bv. `font-weight`, `font-size`).
Style de navigatielijst (`ul`): verwijder `list-style` en default `padding`/`margin`.
Style de navigatielinks (`a`): zet `text-decoration` op `none`, kies een `color`.
Overweeg hoe de items op mobiel staan: misschien `display: inline-block` op de `li`'s met wat `margin-left` ertussen?

Knop Styles:
Definieer de `.btn` class (bv. `padding`, `display: inline-block`, `border-radius`, `text-align: center`).
Definieer `.btn-primary` specifiek voor de hoofdkleur (bv. `background-color`, `color`).

Desktop Layout (Media Query):
Maak een `@media (min-width: ...)` query (bv. 768px).
Selecteer de `.container` binnen de `header`. Gebruik `display: flex` om Flexbox te activeren.
Gebruik `justify-content: space-between` om de logo-div en nav-div uit elkaar te duwen (logo links, nav rechts).
Gebruik `align-items: center` om ze verticaal netjes uit te lijnen.
Selecteer de `nav ul` en maak deze ook een flex-container (`display: flex`, `align-items: center`) om de links en de knop mooi naast elkaar te krijgen en verticaal uit te lijnen.
Pas eventueel de `margin` op de `li`-elementen aan voor meer ruimte op desktop.

Controleer in je browser hoe de header zich gedraagt bij verschillende schermbreedtes.

Stap 2: Homepage - Hero Sectie

Het grote blok bovenaan met titel, tekst en afbeelding.

HTML Structuur (Concept)

Gebruik het <main> element om de hoofdinhoud van je pagina te omsluiten (alles behalve header en footer).

Start deze sectie met een <section> element, geef het een class (bv. `hero`).

Gebruik weer een `div.container` binnen de sectie.

In de container, maak twee hoofdelementen:
Een <div> voor de tekst (class `hero-text`).
Een <div> voor de afbeelding (class `hero-image`).

Vul de `hero-text` div:
Gebruik een <h1> voor de hoofdtitel van de pagina.
Voeg een paragraaf (<p>) toe voor de slogan (geef deze evt. een class `slogan`).
Voeg nog een <p> toe voor de beschrijvende tekst.

Plaats in de `hero-image` div een <img> tag. Vergeet niet een duidelijke `alt`-tekst toe te voegen en de `src` te laten verwijzen naar jouw afbeelding.

CSS Styling (Concept)

Sectie Styles (Mobile):
Geef de `.hero` sectie `padding` en eventueel een `background-color`.
Style de typografie: `h1` (grootte, marge), `.slogan` (grootte, stijl), `p` (regelafstand).

Afbeelding Styles (Mobile):
Zorg dat de afbeelding schaalt: gebruik `max-width: 100%` en `height: auto`.
Gebruik `display: block` om extra witruimte onder de afbeelding te voorkomen.
Geef op mobiel wat `margin-top` aan de afbeelding om ruimte te creëren onder de tekst.

Desktop Layout (Media Query):
In je `@media` query:
Selecteer de `.hero .container`. Pas `display: flex` toe.
Gebruik `align-items: center` voor verticale uitlijning.
Gebruik `gap` (of `margin` op de elementen) om ruimte tussen de tekst- en afbeelding-divs te maken.
Geef de `.hero-text` en `.hero-image` divs een basisgrootte met `flex-basis` (bv. 50% elk, of andere verhouding) of maak ze flexibel met `flex: 1`.
Reset de `margin-top` van de afbeelding die je voor mobiel had ingesteld.

Stap 3: Homepage - Video Sectie

Een sectie om een video te tonen.

HTML Structuur (Concept)

Maak een nieuwe <section> (bv. class `video-section`) binnen de `<main>`.

Optioneel: voeg een titel toe met <h2>.

Plaats de video binnen een `div.container`.

Voor responsieve video's (vooral `iframe`s) is het handig een extra wrapper <div> te gebruiken (class `video-wrapper`) direct om de video heen.

Plaats hierin je <iframe> code (van YouTube/Vimeo) of een HTML5 <video> element met een <source> tag.

CSS Styling (Concept)

Sectie Styles:
Geef de `.video-section` wat `padding`.
Style eventueel de `h2` (bv. `text-align: center`, `margin-bottom`).

Responsive Video Wrapper (Aspect Ratio Trick):
Style de `.video-wrapper`:
    Zet `position: relative`.
    Zet `padding-bottom` met een percentage dat de aspect ratio bepaalt (bv. `56.25%` voor 16:9).
    Zet `height: 0` en `overflow: hidden`.
Style de `iframe` (of `video`) binnen de wrapper:
    Zet `position: absolute`, `top: 0`, `left: 0`.
    Zet `width: 100%`, en `height: 100%`.

Alternatief (Simpeler):
Als de exacte aspect ratio minder belangrijk is:
Laat de wrapper weg.
Style de `iframe` of `video` direct met `max-width: 100%` en `height: auto`.
Centreer eventueel met `display: block` en `margin: auto`.

Stap 4: Homepage - Over Ons Sectie

Lijkt op de Hero, maar met andere content en mogelijk layoutvolgorde.

HTML Structuur (Concept)

Maak weer een <section> (class `about-us`) binnen `<main>`.

Gebruik een `div.container`.

Plaats twee `div`s binnen de container:
Eén voor de afbeelding (`about-image`).
Eén voor de tekst (`about-text`).
Zet ze in de volgorde zoals in het design (bv. afbeelding links, tekst rechts).

Plaats een <img> (met `src` en `alt`) in de `about-image` div.

Plaats een <h2> en een of meer <p> elementen in de `about-text` div.

CSS Styling (Concept)

Sectie Styles (Mobile):
Geef `padding`, eventueel een andere `background-color`.
Style de `h2` en `p` tags.
Zorg dat de afbeelding (`.about-image img`) responsive is (`max-width: 100%`, `height: auto`, `display: block`).
Geef de afbeelding `margin-bottom` voor ruimte op mobiel.

Desktop Layout (Media Query):
Binnen de `@media` query:
Selecteer `.about-us .container`. Pas `display: flex`, `align-items: center`, en `gap` toe.
Gebruik `flex-basis` of `flex: 1` op de `.about-image` en `.about-text` divs om hun breedteverhouding te bepalen.
Verwijder de `margin-bottom` van de afbeelding.

Stap 5: Homepage - Bestemmingen Sectie

Hier tonen we de reisopties als 'kaarten'.

HTML Structuur (Concept)

Maak een <section> (class `destinations`) binnen `<main>`.

Plaats een `div.container` en een <h2> voor de sectietitel.

Maak een wrapper <div> (class `destination-cards`) die alle kaarten bevat.

Voor elke bestemming, gebruik een <article> element (class `card`).

Binnen elke `card`:
    Plaats een <img> (met `src` en `alt`).
    Maak een <div> (class `card-content`) voor de tekst en knop.
    In `card-content`:
        Een <h3> voor de naam.
        Een <p> voor de prijs (class `price`).
        Een <a> tag gestyled als knop (bv. `btn btn-secondary`).

Herhaal de `<article class="card">...</article>` structuur voor alle vier bestemmingen.

CSS Styling (Concept)

Sectie Styles (Mobile):
Geef de sectie `padding` en eventueel `background-color`.
Centreer de `h2` en geef het `margin-bottom`.

Card Styles (Mobile):
Style de `.card`: `background-color` (bv. wit), `border`, `border-radius`.
Geef `margin-bottom` voor ruimte tussen de kaarten (die nu onder elkaar staan).
Gebruik `overflow: hidden` als je `border-radius` op de kaart hebt en de afbeelding bovenaan zit.
Style de `img` binnen de kaart (`max-width: 100%`, `display: block`).
Style de `.card-content` div (`padding`).
Style de `h3`, `.price` (maak deze opvallend met `font-weight`, `color`, `font-size`).
Style de `.btn-secondary` (misschien een andere achtergrondkleur dan `.btn-primary`).

Desktop Layout (Media Queries):
    <em>Tablet / Small Desktop (bv. `@media (min-width: 600px)`):</em>
        Selecteer de `.destination-cards` wrapper. Pas `display: flex` toe.
        Cruciaal: gebruik `flex-wrap: wrap` zodat kaarten naar de volgende regel kunnen springen.
        Gebruik `gap` voor ruimte tussen kaarten.
        Selecteer de `.card`. Gebruik `flex-basis: calc(50% - helft_van_gap)` om twee kaarten per rij te krijgen.
        Behoud `margin-bottom` voor het geval van een oneven aantal kaarten.

    <em>Wider Desktop (bv. `@media (min-width: 992px)`):</em>
        Selecteer de `.card` opnieuw binnen deze query.
        Pas de `flex-basis` aan voor vier kolommen, bv. `calc(25% - aangepaste_gap_berekening)`.
        Experimenteer eventueel met simpele percentages en `justify-content` op de container.

Stap 6: Homepage - Footer

De afsluitende balk onderaan.

HTML Structuur (Concept)

Gebruik het semantische <footer> element na het `<main>` element.

Plaats er een `div.container` in.

Voeg een <p> element toe met je copyright informatie (bv. `&copy; 2024 Naam - Klas - School`).

CSS Styling (Concept)

Style de `footer`:
Geef het een `background-color`, `color` (voor de tekst), `text-align: center`, en `padding`.
Geef het eventueel `margin-top` om het los te maken van de content erboven.
Style de `p` binnen de footer (bv. `margin: 0` om extra ruimte te voorkomen).
Deze styling is vaak hetzelfde voor mobiel en desktop.

Homepage klaar (conceptueel)! Vul nu alles met echte content.

Stap 7: Blogpagina (`blog.html`) - Structuur

Deze pagina hergebruikt de header en footer.

HTML Structuur (Concept)

Hergebruik Header/Footer:
Kopieer de volledige HTML van je `<header>` en `<footer>` uit `index.html` naar `blog.html`.

Maak een <main> element voor de blog content.

Afbeeldingen Sectie:
    Maak een <section> (class `blog-images`).
    Plaats een `div.container`.
    Maak twee <div> elementen (class `image-placeholder`).
    Plaats in elke div een <img> met `src` en `alt` voor je blogafbeeldingen.

Lijsten Sectie:
    Maak een <section> (class `blog-lists`).
    Plaats een `div.container`.
    Maak drie aparte blokken, bijvoorbeeld met <div> (class `list-block`).
    In elk blok:
        Een <h2> met de titel van de lijst ("5 Dingen geleerd...", etc.).
        Daaronder een genummerde lijst (<ol>).
        Vul de lijst met <li> elementen waarin je daadwerkelijk je punten noteert.

Stap 8: Blogpagina (`blog.html`) - CSS Styling

Voeg deze stijlen toe aan je bestaande style.css.

CSS Styling (Concept)

Afbeeldingen Sectie (Mobile):
Style `.blog-images` (bv. `padding`, `background-color`).
Zorg dat de afbeeldingen (`.image-placeholder img`) responsive zijn (`max-width: 100%`, etc.).
Geef ze `margin-bottom`.

Lijsten Sectie (Mobile):
Style `.blog-lists` (`padding`).
Style `.list-block` (`margin-bottom` om ze uit elkaar te houden).
Style de `h2` binnen de lijsten (bv. `font-size`, `border-bottom`).
Style de `ol` en `li` elementen (standaard `padding-left` op `ol` is vaak ok, geef `li` wat `margin-bottom` en/of `line-height`).

Desktop Layout (Media Query):
    Selecteer `.blog-images .container`. Pas `display: flex` en `gap` toe.
    Geef de `.image-placeholder` divs een `flex-basis` van `calc(50% - half_gap)` om ze naast elkaar te krijgen.
    Reset de `margin-bottom` op de afbeeldingen.
    De lijsten sectie (`.blog-lists`) blijft waarschijnlijk prima onder elkaar staan.

Stap 9: Afronden en Controleren

De laatste loodjes!

  1. Content! Vul écht alle teksten in, plaats je gevonden afbeeldingen en video. Schrijf zinvolle alt-teksten. Vul de lijsten op de blogpagina.
  2. Testen, Testen, Testen: Check beide pagina's op verschillende schermbreedtes (gebruik de developer tools van je browser!). Werkt de responsive layout? Ziet alles er netjes uit? Geen rare overlappingen of te kleine tekst?
  3. Links: Werkt de navigatie tussen de homepage en blogpagina?
  4. Validatie (Bonus): Controleer je HTML en CSS op fouten met online validators.
  5. Video Maken: Neem je scherm op. Laat de werkende site zien (ook responsive!). Loop kort door je HTML en CSS: leg uit *waarom* je bepaalde tags en properties hebt gebruikt (semantiek, Flexbox, mobile-first). Focus op begrip. Max 5 minuten.
  6. Inpakken: Maak een .zip van je projectmap.
  7. Inleveren: Dien het .zip bestand en je video in.

Top! Met deze conceptuele gids heb je de handvatten om de website zelfstandig op te bouwen en te begrijpen wat je doet. Focus op de structuur, de semantiek en de logica van CSS en Flexbox. Succes!
