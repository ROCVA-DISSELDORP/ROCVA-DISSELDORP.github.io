📋 Over deze hackathon

Welkom bij de ROCVA Hackathon! Je hebt twee coole challenges waar je uit kunt kiezen. Kies datgene wat je het meest aanspreekt en ga aan de slag. 🎮 Challenge 1: Game Development (Unity of THREE.js) ▼

Bouw een coole game! Kies tussen Unity of THREE.js (WebGL). Ga alleen (of multiplayer, nog beter!) aan de slag en maak iets wat spelersgeluk geeft.

📚 User Stories
Story 1: Als speler wil ik de game kunnen starten zodat ik kan spelen
Story 2: Als speler wil ik karakters/objecten zien bewegen zodat het voelt als een echte game
Story 3: Als speler wil ik punten/score verdienen zodat ik zie hoe goed ik speel
Story 4: Als speler wil ik een game-over scherm zien zodat ik weet wanneer het voorbij is
Story 5: Als speler wil ik zo mogelijk tegen vrienden spelen (optional multiplayer lees meer)
📁 Bestandstructuur
game-project/
├── index.html                 (hoofdpagina)
├── css/
│   └── style.css              (styling)
├── js/
│   ├── main.js                (gameloop en initialisatie)
│   ├── player.js              (spelerlogica)
│   ├── enemy.js               (vijandlogica)
│   ├── collision.js           (botsing detectie)
│   └── ui.js                  (score, UI updates)
├── assets/
│   ├── models/                (3D modellen voor THREE.js)
│   ├── textures/              (afbeeldingen)
│   └── sounds/                (geluidseffecten)
└── package.json               (dependencies)
🔧 Pseudocode - Game Loop
function gameLoop() { // Input verwerken (toetsenbord, muis, controller) handlePlayerInput() // Update game state updatePlayerPosition() updateEnemies() updateProjectiles() // Detecteer botsingen checkCollisions() // Update score/UI updateUI() // Render alles render() // Request volgende frame requestAnimationFrame(gameLoop) }

🎬 Entertainment APIs

📚 User Stories

Story 2: Als gebruiker wil ik zoeken naar items zodat ik specifieke data vindt

🔧 Pseudocode - Data Fetching
// API service async function fetchFromAPI(endpoint) { try { const response = await fetch(`${API_BASE_URL}/${endpoint}`) if (!response.ok) throw new Error('API error') return await response.json() } catch (error) { console.error('Fetch error:', error) return null } } // Component logica function App() { const [items, setItems] = useState([]) const [loading, setLoading] = useState(false) const searchItems = async (query) => { setLoading(true) const data = await fetchFromAPI(`search?q=${query}`) setItems(data) setLoading(false) } return ( <div> <SearchBar onSearch={searchItems} /> {loading && <p>Laden...</p>} <ItemList items={items} /> </div> ) }

💡 Handige Tips voor Studenten

🎯 Structuur en Planning

  • Begin klein: Zorg dat de basis werkt voordat je fancy features toevoegt
  • Gebruik de backlog: Werk de taken stap voor stap af, check ze af
  • Test regelmatig: Zorg dat je code na elke wijziging nog werkt
  • Git commits: Commit je werk regelmatig met duidelijke berichten

🤖 AI Slim Gebruiken

✅ Zo vraag je goed aan AI:
  • "Hoe maak ik een zoekveld in React dat realtime filtert?"
  • "Geef me een voorbeeld van hoe ik data van [API] fetch"
  • "Hier is mijn code: [CODE]. Waarom werkt dit niet?"
  • "Hoe kan ik deze functie verbeteren/optimaliseren?"
❌ Zo vraag je niet aan AI:
  • "Maak mijn hele app voor me" (dan leer je niets!)
  • "Dit werkt niet" (AI kan niet helpen zonder meer info)
  • AI gebruiken zonder de code zelf te begrijpen

📚 HTML/Tailwind Tutorials Genereren

Super handige prompt voor Tailwind layouts:
"Maak een responsive layout met Tailwind CSS voor een [beschrijf jouw idea]. Zorg dat het mooi eruit ziet op telefoon, tablet en desktop. Gebruik alleen Tailwind classes, geen custom CSS."
Prompt voor HTML5 semantic code:
"Schrijf semantische HTML5 voor een [beschrijf pagina]. Gebruik header, nav, main, section, article, footer tags. Maak het optimaal voor SEO."
Prompt voor animaties:
"Maak een animatie met Tailwind CSS voor [beschrijf wat je wil animeren]. Zorg dat het smooth en performant is. Gebruik Tailwind's animation classes."

🔍 Debugging Tips

  • Console Log: Gebruik console.log() om je code stap voor stap te debuggen
  • Browser DevTools: Leer de F12 developer tools kennen (Elements, Console, Network tabs)
  • Error messages lezen: Lees error messages goed, ze vertellen je wat er mis is!
  • Stapje teruggaan: Als iets breekt, ga terug naar de vorige versie en werk langzamer

🎨 Design Tips

  • Tailwind Classes: Leer de basics: grid, flexbox, padding, margin, colors
  • Whitespace: Goed gebruik van ruimte maakt design beter leesbaar
  • Consistent Kleuren: Kies 2-3 hoofdkleuren en hou je daaraan vast
  • Responsive: Test je app op telefoon, tablet EN desktop

🏆 Hoe Win Je Deze Hackathon?

De winnaar is niet wie het meeste code schrijft, maar wie het BESTE resultaat levert EN het kan uitleggen!

  • Werkende app: Je project moet daadwerkelijk functioneren zonder crashes
  • Afgewerkt design: Het ziet er professioneel en mooi uit
  • User experience: De app is makkelijk te gebruiken en voelt responsief
  • Originele idee/twist: Je hebt iets unieks of creatiefs gemaakt
  • Code kwaliteit: Je code is schoon, begrijpelijk en goed georganiseerd
  • Je kunt uitleggen wat je hebt gedaan: Je begrijpt je eigen code en kunt het demonstreren
  • Extra features (bonus): Bonus punten voor features buiten de basiseisen
