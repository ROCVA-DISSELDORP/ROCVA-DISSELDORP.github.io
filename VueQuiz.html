Jouw Eerste Nuxt.js Quiz App

Een complete, stap-voor-stap handleiding voor het bouwen van een frontend applicatie met Nuxt.js, Vue.js en Tailwind CSS.

Welkom! In deze tutorial bouwen we samen de quiz-opdracht van begin tot eind. Dit is waarschijnlijk je eerste kennismaking met een JavaScript-framework, en dat kan best spannend zijn. We nemen alles rustig door, leggen elk nieuw concept uit en verwijzen je naar de officiële documentatie voor als je meer wilt weten.

Wat gaan we gebruiken?

  • Nuxt.js: Een framework dat bovenop Vue.js is gebouwd. Het neemt veel ingewikkelde zaken voor je uit handen, zoals het opzetten van pagina's (routing) en het efficiënt ophalen van data.
  • Vue.js: Een modern JavaScript-framework waarmee we interactieve gebruikersinterfaces bouwen. We focussen op de basis: componenten en "reactiviteit".
  • Tailwind CSS: Een utility-first CSS-framework. In plaats van zelf CSS-bestanden te schrijven, gebruiken we kant-en-klare klassen direct in onze HTML.

Stap 1: Project Opzetten

We beginnen met het aanmaken van een nieuw Nuxt-project en het installeren van Tailwind CSS.

1.1 Een nieuw Nuxt-project aanmaken

Open je terminal (bijvoorbeeld Git Bash, PowerShell of de terminal in VSCode) en voer het volgende commando uit. Dit maakt een nieuwe map `quiz-app` aan met alle benodigde startbestanden. Ook voegt dit direct de ui library van Nuxt toe wel zo makkelijk 

npm create nuxt@latest -- -t [naam-van-je-project]

Alles installeren duurt wel even (5 min) en je moet een paar vragen beantwoorden. Kies voor: npm, yes initialize git project, yes install modules, gebruik spatie en pijltjes en selecteer: [eslint, icon, image, ui]. Gebruik enter om naar de volgende stap te gaan.

Navigeer naar de zojuist aangemaakte projectmap:

cd [naam-van-je-project]

Installeer nu alle benodigde pakketten (dependencies) die in `package.json` staan. Dit doet npm (Node Package Manager) voor je. 

npm install

1.2 Tailwind CSS installeren en configureren

Nuxt heeft een speciale "module" die Tailwind CSS naadloos integreert. We installeren deze via de terminal. De `-D` vlag betekent dat het een 'development dependency' is: een tool die we alleen tijdens het ontwikkelen nodig hebben. 

npm install -D @nuxtjs/tailwindcss

Open nu het bestand `nuxt.config.ts`. Dit is het centrale configuratiebestand voor je Nuxt-project. Voeg `@nuxtjs/tailwindcss` toe aan de `modules` array. 

// nuxt.config.ts
export default defineNuxtConfig({
  devtools: { enabled: true },
  modules: [
    '@nuxtjs/tailwindcss'
  ]
})

1.3 De ontwikkelserver starten

Je project is nu klaar. Start de ontwikkelserver om je werk live in de browser te zien. 

npm run dev

Open de URL die in je terminal verschijnt (meestal http://localhost:3000). Je ziet nu de standaard welkomstpagina van Nuxt.

Stap 2: De Welkomstpagina (`index.vue`)

Nuxt gebruikt een `pages` map om automatisch webpagina's (routes) aan te maken. Een bestand `pages/index.vue` wordt de hoofdpagina (`/`), en `pages/quiz.vue` wordt de `/quiz` pagina.

Concept: Single File Components (SFCs)

In Vue werken we met `.vue` bestanden. Dit zijn "Single File Components". Zo'n bestand combineert de structuur, logica en styling van een component op één plek:

  • <template>: Hier schrijf je de HTML-structuur.
  • <script setup>: Hier schrijf je de JavaScript-logica (volgens de moderne Vue 3 Composition API).
  • <style>: Hier kun je component-specifieke CSS schrijven.

Lees meer in de officiële Vue documentatie over SFC's.

Verwijder het standaard `app.vue` bestand en maak een map `pages` aan. Maak in die map een nieuw bestand: `pages/index.vue`. Plak hier de volgende code in.

Stap 3: Naam Opslaan en Navigeren

Hoe onthouden we de naam van de speler op andere pagina's? Hiervoor gebruiken we een krachtig concept van Nuxt: useState.

Concept: useState

useState is een functie (een "composable") in Nuxt die een reactieve, gedeelde staat creëert. "Reactief" betekent dat de interface automatisch update als de waarde verandert. "Gedeeld" betekent dat de staat toegankelijk is in al je componenten en pagina's. Perfect om data zoals een gebruikersnaam of score over de hele applicatie te onthouden.

Lees meer in de officiële Nuxt documentatie over useState.

3.1 Naam binden en opslaan

We updaten `pages/index.vue` om de ingevoerde naam op te slaan en de gebruiker door te sturen naar de quizpagina. Bestudeer de commentaren in de code. 

<!-- In je <template>, voeg v-model en @click toe -->
<input 
  v-model="playerName"
  @keyup.enter="startQuiz" <!-- Bonus: start quiz ook met Enter-toets -->
  type="text" 
  placeholder="Jouw naam" 
  class="border p-3 rounded w-full mb-4 focus:outline-none focus:ring-2 focus:ring-blue-500"
/>
<button @click="startQuiz" class="...">Start de Quiz</button>

<!-- Update je <script setup> -->
<script setup>
// 1. Importeer de benodigde functies uit Vue en Nuxt
import { ref } from 'vue'
import { useRouter } from '#app' // of 'vue-router'

// 2. Creëer een globale state voor de spelersnaam. 
// De eerste parameter ('playerName') is een unieke sleutel.
// De tweede is een functie die de beginwaarde retourneert.
const user = useState('playerName', () => '')

// 3. 'ref' maakt een lokale, reactieve variabele. We koppelen deze met v-model aan het inputveld.
const playerName = ref('')

// 4. 'useRouter' geeft ons toegang tot de router, waarmee we kunnen navigeren.
const router = useRouter()

// 5. Deze functie wordt aangeroepen als op de knop wordt geklikt.
function startQuiz() {
  if (playerName.value.trim()) {
    // Sla de ingevulde naam op in de globale state.
    user.value = playerName.value
    // Stuur de gebruiker door naar de /quiz pagina.
    router.push('/quiz')
  } else {
    alert('Voer alsjeblieft een naam in.')
  }
}
</script>
Belangrijk: Maak nu alvast een leeg bestand `pages/quiz.vue` aan. Als je dit niet doet, krijg je een 404 (Not Found) foutmelding als je probeert te navigeren.

Stap 4: Quizdata Ophalen uit JSON

De vragen en antwoorden horen in een `data.json` bestand. We plaatsen dit in de `public` map. Bestanden in deze map zijn direct toegankelijk in de browser.

4.1 Maak het `data.json` bestand

Maak een map `public` in de hoofdmap van je project. Maak daarin een bestand `data.json` met deze inhoud: 

// public/data.json
{
  "questions": [
    {
      "question": "Welk framework gebruiken we voor deze tutorial?",
      "image": "https://nuxtjs.org/meta_400.png",
      "answers": [

        { "text": "Angular", "correct": false },

Concept: useFetch

useFetch is dé manier in Nuxt om data op te halen, bijvoorbeeld uit een JSON-bestand of een externe API. Het grote voordeel is dat Nuxt dit al op de server kan doen, waardoor je data direct beschikbaar is als de pagina laadt. Dit maakt je website sneller en beter voor zoekmachines.

Lees meer in de officiële Nuxt documentatie over useFetch.

4.2 Data ophalen in de quizpagina

Open je (nu nog lege) `pages/quiz.vue` en gebruik `useFetch` om de vragen op te halen. 

<template>
  <div class="p-8">
    <h1 class="text-2xl font-bold">Quiz Pagina</h1>
    <p>Speler: {{ user }}</p>
    
    <!-- We tonen de data hier even om te testen. Dit is een goede manier om te debuggen! -->
    <pre v-if="quizData">{{ quizData.questions }}</pre>
    <p v-else>Quiz wordt geladen...</p>
  </div>
</template>

<script setup>
// Haal de opgeslagen naam weer op uit de state.
const user = useState('playerName')

// Haal de quizdata op. De 'await' zorgt ervoor dat we wachten tot de data binnen is.
const { data: quizData } = await useFetch('/data.json')
</script>

Als je nu naar de quizpagina navigeert, zie je de ruwe data van je JSON-bestand op het scherm. Dit betekent dat het ophalen is gelukt!

Stap 5: De Quiz Pagina Bouwen

Dit is de kern van de applicatie. We moeten de vragen een voor een tonen, de antwoorden shufflen en de timer implementeren.

Concepten: computed, v-if en v-for

computed: Een 'computed property' is een waarde die wordt afgeleid van andere reactieve data. Het grote voordeel is dat Vue deze waarde onthoudt (cached) en alleen opnieuw berekent als een van de afhankelijkheden verandert. We gebruiken dit om de huidige vraag en de geschudde antwoorden te bepalen.
Lees meer over Computed Properties.

v-if en v-for: Dit zijn "directives" in Vue. Met v-if kun je elementen conditioneel tonen (alleen als een voorwaarde waar is). Met v-for kun je door een lijst (array) loopen en voor elk item een HTML-element renderen.
Lees meer over v-if en v-for.

Hieronder staat een meer complete opzet voor pages/quiz.vue. Dit is een grote stap, dus neem de tijd om de commentaren in de code te lezen. 

<template>
  <div class="min-h-screen bg-gray-100 flex items-center justify-center p-4">
    <!-- Gebruik v-if om te wachten tot de huidige vraag beschikbaar is -->
    <div v-if="currentQuestion" class="bg-white p-8 rounded-lg shadow-md w-full max-w-2xl">
      <div class="flex justify-between items-center mb-4">
        <h2 class="text-xl font-semibold">Speler: {{ user }}</h2>
        <div class="text-2xl font-bold bg-blue-600 text-white rounded-full h-16 w-16 flex items-center justify-center">
          15 <!-- Timer placeholder -->
        </div>
      </div>

      <p class="text-gray-500 mb-4">Vraag {{ currentQuestionIndex + 1 }} van {{ quizData.questions.length }}</p>
      <h3 class="text-2xl mb-4 font-semibold">{{ currentQuestion.question }}</h3>
      <img v-if="currentQuestion.image" :src="currentQuestion.image" alt="Vraag afbeelding" class="mb-6 rounded-lg max-h-64 mx-auto">

      <div class="grid grid-cols-1 md:grid-cols-2 gap-4">
        <!-- We gebruiken v-for om door de antwoorden te loopen. :key is belangrijk voor Vue. -->
        <button 
          v-for="(answer, index) in shuffledAnswers" 
          :key="index"
          class="bg-gray-200 p-4 rounded-lg text-left text-lg hover:bg-gray-300 transition">
          {{ answer.text }}
        </button>
      </div>
    </div>
    <div v-else class="text-center">
        <p class="text-xl">Quiz aan het voorbereiden...</p>
    </div>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

const user = useState('playerName')
const { data: quizData } = await useFetch('/data.json')

// 'ref' om de index van de huidige vraag bij te houden.
const currentQuestionIndex = ref(0)

// Een computed property die de data van de huidige vraag retourneert.
const currentQuestion = computed(() => {
  // Zorg ervoor dat quizData en questions bestaan voordat je ze benadert.
  return quizData.value?.questions[currentQuestionIndex.value]
})

// Een computed property die de antwoorden voor de huidige vraag shufflet.
const shuffledAnswers = computed(() => {
  if (currentQuestion.value) {
    // Maak een kopie met [...] en shuffle deze met .sort()
    return [...currentQuestion.value.answers].sort(() => Math.random() - 0.5)
  }
  return []
})

// Puzzelstukjes die je zelf nog moet invullen:
// 1. Logica voor de timer (zie de opdracht over Lifecycle Hooks).
// 2. Een functie die wordt aangeroepen als op een antwoord wordt geklikt.
//    - Deze functie moet het antwoord controleren.
//    - Het antwoord opslaan.
//    - Naar de volgende vraag gaan (currentQuestionIndex.value++).
//    - Als er geen vragen meer zijn, navigeren naar de scorepagina.
</script>

Refresh de quizpagina. Je zou nu de eerste vraag moeten zien met de antwoorden in een willekeurige volgorde. Refresh nog een paar keer om te zien dat de volgorde steeds verandert!

Stap 6: De Score Pagina & Laatste Puzzelstukjes

De laatste stap is het tonen van de resultaten. Hiervoor heb je een nieuwe `useState` nodig om de antwoorden van de gebruiker in op te slaan tijdens de quiz.

6.1 Antwoorden Opslaan

Definieer in `pages/quiz.vue` (of een aparte `composables/` map) een nieuwe state: 

const userAnswers = useState('userAnswers', () => [])

Wanneer een gebruiker een antwoord kiest, voeg je een object toe aan deze array. Bijvoorbeeld: 

userAnswers.value.push({
  question: currentQuestion.value.question,
  userAnswer: 'Het gekozen antwoord',
  correctAnswer: 'Het juiste antwoord',
  isCorrect: true // of false
})

6.2 De Scorepagina Bouwen

Maak een bestand `pages/score.vue`.

<template>
  <div class="min-h-screen bg-gray-100 p-4 md:p-8">
    <div class="bg-white p-8 rounded-lg shadow-md w-full max-w-3xl mx-auto">
      <h1 class="text-3xl font-bold mb-4">Resultaten voor {{ user }}</h1>
      <p class="text-xl mb-8">Je eindscore: <span class="font-bold">{{ score }} / {{ userAnswers.length }}</span></p>

      <div class="space-y-4">
        <!-- Loop door de gegeven antwoorden -->
        <div v-for="(result, index) in userAnswers" :key="index" class="border-t pt-4">
          <h3 class="text-lg font-semibold">{{ result.question }}</h3>
          <p>Jouw antwoord: 
            <span class="font-bold" :class="result.isCorrect ? 'text-green-600' : 'text-red-600'">
              {{ result.userAnswer }}
            </span>
          </p>
          <p v-if="!result.isCorrect">Correcte antwoord: 
            <span class="font-bold text-green-600">{{ result.correctAnswer }}</span>
          </p>
        </div>
      </div>

    </div>
  </div>
</template>

<script setup>
import { computed } from 'vue'

const user = useState('playerName')
const userAnswers = useState('userAnswers', () => [])

// Berekent de score met een computed property
const score = computed(() => {
  return userAnswers.value.filter(answer => answer.isCorrect).length
})
</script>
