The Ultimate PHP Course: From Beginner to Pro

An extensive, step-by-step guide to mastering PHP for web development. We'll start with the absolute basics and end with the skills needed to build modern, database-driven applications.

Part 1: Foundations

1.1 What is PHP & Its Role?

PHP is a server-side language. Its primary job is to:

  1. Process Input: Receive data from a user's form submission or a URL.
  2. Work with Data: Communicate with a database to create, read, update, or delete information.
  3. Create Output: Generate a final HTML page to send back to the user's browser.

1.2 Setup (XAMPP)

To run PHP, you need a server environment. XAMPP is a free package that includes Apache (web server), MySQL (database), and PHP. Place your projects in the `htdocs` directory and view them at `http://localhost/your-project-folder/`.

1.3 Basic Syntax & Variables

PHP code lives in `` tags. Statements end with a semicolon `;`. Variables must begin with a `$` sign. 

<?php
// This is a comment
echo "Hello, World!"; // echo prints output

$greeting = "Welcome";      // String
$userCount = 100;         // Integer
$isLoggedIn = true;   // Boolean

// Join strings with a dot (.)
echo $greeting . ", you are user #" . $userCount;
?>

1.4 Error Reporting (Crucial for Development)

By default, PHP might hide errors. When developing, you MUST turn them on to see what's wrong. Add this to the top of your main PHP file. 

<?php
ini_set('display_errors', 1);
error_reporting(E_ALL);
?>

Part 2: Data Structures & Control Flow

2.1 Arrays

Arrays store multiple values. Associative arrays, with named keys, are the most common type for structured data. 

<?php
// Associative Array
$user = [
    'name' => 'John Doe',
    'email' => 'john@example.com',
];
echo $user['email'];
?>

2.2 Control Structures

These allow your code to make decisions (`if`/`else`) and repeat actions (`foreach`). 

<?php
$isLoggedIn = true;
if ($isLoggedIn) {
    echo "Welcome back!";
} else {
    echo "Please log in.";
}
?>

2.3 PHP Templating

PHP is often mixed with HTML. For cleaner code, you can use an alternative syntax for control structures. 

<?php
$tasks = ['Task 1', 'Task 2', 'Task 3'];
?>

<h2>My Tasks</h2>
<ul>
    <?php foreach ($tasks as $task): ?>
        <li><?= $task ?></li> <-- <?= is a shortcut for <?php echo -->
    <?php endforeach; ?>
</ul>

<?php if (empty($tasks)): ?>
    <p>No tasks yet!</p>
<?php endif; ?>

Part 3: Reusing & Organizing Code

3.1 Functions

Functions are reusable blocks of code. They help keep your code clean and prevent repetition.

3.2 Building a Basic Template

You can create reusable HTML parts like a header and footer. Use `include_once` to pull them into your pages.

`header.php`

<!DOCTYPE html>
<html>
<head><title>My Website</title></head>
<body>
    <header><h1>My Awesome Site</h1></header>

`footer.php`

<footer>
    <p>Copyright <?= date('Y') ?></p>
</footer>
</body>
</html>

`index.php`

<?php include_once 'header.php'; ?>

<main>
    <p>This is the main content of the page.</p>
</main>

<?php include_once 'footer.php'; ?>

Part 4: Handling User Interaction

This is where PHP shines: processing form data, managing user sessions, and interacting with the browser.

4.1 Superglobals: `$_GET` and `$_POST`

PHP provides special "superglobal" variables to access web data. They are available in any scope.

  • $_GET: An associative array of variables passed in the URL's query string (e.g., `page.php?id=123`).
  • $_POST: An associative array of variables passed from a submitted HTML form with `method="POST"`. 
<!-- form.html -->
<form action="process.php" method="POST">
    <input type="text" name="username">
    <button type="submit">Submit</button>
</form>

<!-- process.php -->
<?php
if (isset($_POST['username'])) {
    $name = htmlspecialchars($_POST['username']);
    echo "Hello, " . $name;
}
?>

4.2 Sessions

Sessions are used to store user information across multiple pages (e.g., keeping a user logged in). The data is stored on the server, not the browser. 

<?php
// Must be at the VERY TOP of the file
session_start();

// Storing a session variable (e.g., after login)
$_SESSION['user_id'] = 123;

// Accessing it on another page
if (isset($_SESSION['user_id'])) {
    echo "You are logged in!";
}

// To log out
session_destroy();
?>

Part 5: Working with Databases (MySQLi)

This is the core of any dynamic web application. We'll use the `mysqli` (MySQL Improved) extension.

5.1 Connecting to MySQL

<?php
$db_host = 'localhost';
$db_user = 'root';
$db_pass = ''; // Default for XAMPP
$db_name = 'my_database';

$conn = new mysqli($db_host, $db_user, $db_pass, $db_name);

// Always check the connection
if ($conn->connect_error) {
    die("Connection Failed: " . $conn->connect_error);
}
?>

5.2 The Secure Way: Prepared Statements

To prevent a major vulnerability called SQL Injection, you must use prepared statements for any query that includes user data. This is non-negotiable for professional code.

Fetching Data (SELECT)

<?php
$sql = "SELECT * FROM users WHERE id = ?";
$stmt = $conn->prepare($sql);
$stmt->bind_param("i", $userId); // "i" for integer
$stmt->execute();
$result = $stmt->get_result();
$user = $result->fetch_assoc();
?>

Inserting Data (INSERT)

<?php
$sql = "INSERT INTO users (username, email) VALUES (?, ?)";
$stmt = $conn->prepare($sql);
$stmt->bind_param("ss", $username, $email); // "s" for string
$stmt->execute();
?>

Part 6: Essential Security Practices

Writing secure code is not optional. These are the absolute minimum practices you must follow.

6.1 Preventing XSS with `htmlspecialchars()`

Whenever you print data that a user has submitted, wrap it in `htmlspecialchars()`. This prevents users from injecting malicious HTML or JavaScript into your pages. 

echo htmlspecialchars($userComment);

6.2 Secure Password Handling

NEVER store passwords as plain text. Use PHP's built-in functions to securely hash and verify passwords. 

<?php
// Hashing a password during registration
$passwordFromUser = 'user-password123';
$hashedPassword = password_hash($passwordFromUser, PASSWORD_DEFAULT);
// Store $hashedPassword in your database

// Verifying a password during login
$passwordFromDb = /* fetch the hash from the DB */;
if (password_verify($passwordFromUser, $passwordFromDb)) {
    echo 'Password is correct!';
}
?>

Part 7: The Path to Modern PHP

7.1 Introduction to Object-Oriented PHP (OOP)

OOP organizes code into reusable blueprints (`classes`) for creating objects. This is the dominant paradigm in modern PHP.

7.2 Classes and Objects

A class is a blueprint (e.g., `User`). An object is an instance of that class (e.g., a specific user like `$john`). 

<?php
      class User {
        // Properties (class variables)
        public $username;
        private $email; // private means it can only be accessed from inside this class
      
        // Constructor (runs when a new object is created)
        public function __construct($username, $email) {
            $this->username = $username;
            $this->email = $email;
        }
      
        // Method (class function)
        public function sayHello() {
            return "Hello, my name is " . $this->username;
        }
      }
      
      // Create an object (an instance of the User class)
      $user1 = new User('JohnDoe', 'john@example.com');
      echo $user1->sayHello();
      ?>

Practical Example: A Database Class

A great use for a class is to wrap up your database connection logic so it's easy to reuse and manage. 

<?php
class Database {
    public $connection;
    public function __construct($host, $user, $pass, $name) {
        $this->connection = new mysqli($host, $user, $pass, $name);
        if ($this->connection->connect_error) {
            die("Connection Failed");
        }
    }
}
$db = new Database('localhost', 'root', '', 'my_db');
// Now you can pass the $db->connection object around.
?>

7.2 Composer: The PHP Package Manager

Composer is an essential tool for managing third-party libraries (packages). It handles downloading, updating, and autoloading classes for you.

7.3 The Bridge to Frameworks: MVC

Most modern PHP frameworks (like Laravel and Symfony) use the **Model-View-Controller (MVC)** design pattern:

  • Model: The data layer. Interacts with your database (e.g., a `User` model that can fetch users).
  • View: The presentation layer. The HTML templates that the user sees.
  • Controller: The logic layer. It receives requests, uses the Model to get data, and passes that data to the View to be rendered.

Understanding this separation of concerns is the final step before you're ready to learn a framework.
